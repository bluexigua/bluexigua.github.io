Hexo + Github Pages 搭建个人博客

背景

工作之后发现，其实写文档的能力挺重要的，所以希望记录下平常学技术过程中碰到的问题还有学过的东西，毕竟好记性不如烂笔头。为啥要整个博客写呢？因为好zhuang玩bi 👀

前言

做了一些相关的调研，最终选用 Hexo + Github Pages 架构的原因是，省钱省力省时，并且满足了我的需求，不需要花钱买服务器，也不用复杂的部署服务，同时还实现了一个简单个人博客所需要的所有功能：支持markdown和简单的文件归档。
安装环境：Mac M1芯片

安装步骤

其实安装东西，一般对着官方文档来进行是最稳的，所以我也会将每步对应的官方文档的链接贴出。

Step 0：

之所以将 Node.js 的安装和 git ssh 的配置放在 Step0, 是因为这些都比较常见，可能之前就已经配置过了

安装 Node.js

可以使用 node -v 和 npm -v 查看之前是否已经安装过了。
没有的话，可以访问 官网 下载，然后安装即可，最后会提示安装路径和需要配置的环境变量：

安装 git 配置 ssh

Git官方下载
SSH连接GitHud

Step1

安装Hexo

mac 需要用管理员权限安装: sudo -s
PS: 如果挂了VPN这里需要关掉代理，emmmmm 我被这个问题卡了好久😑
官方安装文档

  • 文件夹目录
    • _config.yml: 填写网站的配置信息
    • scaffolds: 模版文件夹。 _config.yml 中的 default_layout 指定默认使用的默认模版，或者使用 hexo new ${模版名} "${文章名}" 指定生成新文件使用的模版，Hexo 会根据 scaffold 来建立文件。
    • themes: 主题文件夹；Hexo根据主题生成静态页面。

至此，已经可以本地启动，访问本地的静态页面了：

  1. hexo server
  2. 访问网址： http://localhost:4000/
